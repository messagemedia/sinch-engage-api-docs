# API Reference

Welcome to the MessageMedia API reference documentation. This documentation provides detailed information about the available API endpoints, request/response formats, and authentication methods.

## Base URLs

- **APAC Production**: `https://api.messagemedia.com`
- **EU Production**: `https://eu.app.api.sinch.com`

## Authentication

All API endpoints require authentication. You can authenticate your requests using either Basic Authentication or HMAC Signature Authentication.

## Error Handling

All error responses follow this format:

```json
{
  "error": {
    "code": "error_code",
    "message": "Human-readable error message",
    "details": "Additional error details (if available)"
  }
}
```

### Common Error Codes

| Status Code | Error Code | Description |
|-------------|------------|-------------|
| 400 | `invalid_request` | The request was invalid or cannot be served |
| 401 | `unauthorized` | Authentication failed or not provided |
| 403 | `forbidden` | Not authorized to access the resource |
| 404 | `not_found` | The requested resource doesn't exist |
| 429 | `rate_limit_exceeded` | Rate limit exceeded |
| 500 | `internal_server_error` | Internal server error |

## Webhook Security

When implementing webhooks, verify the `X-MessageMedia-Signature` header to ensure the request is from MessageMedia. The signature is generated using HMAC-SHA1 with your API secret as the key.
