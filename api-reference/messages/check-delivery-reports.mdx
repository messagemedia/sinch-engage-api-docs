# Check Delivery Reports

Retrieve delivery reports for messages that have been sent.

<Endpoint
  method={MethodTypes.GET}
  path="/v1/delivery_reports"
  auth={false}
  beta={false}
>
  <Method
    type={MethodTypes.GET}
    path="/v1/delivery_reports"
    summary="Check delivery reports"
    id="checkDeliveryReports"
    auth={false}
  />
</Endpoint>

## Query Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `start_date` | String | No | Start date for filtering reports (ISO 8601 format) |
| `end_date` | String | No | End date for filtering reports (ISO 8601 format) |
| `status` | String | No | Filter by delivery status (e.g., 'delivered', 'failed') |
| `limit` | Integer | No | Maximum number of reports to return (default: 100, max: 1000) |
| `offset` | Integer | No | Offset for pagination (default: 0) |

## Example Request

```bash
curl -X GET \
  'https://api.messagemedia.com/v1/delivery_reports?start_date=2023-01-01T00:00:00Z&limit=10' \
  -H 'Authorization: Basic YOUR_API_KEY'
```

## Example Response

```json
{
  "delivery_reports": [
    {
      "message_id": "4fb9e5a3-2091-4d5c-8e5d-8e8c1b9b3b8b2",
      "status": "delivered",
      "destination_number": "+61412345678",
      "timestamp": "2023-01-01T12:00:00Z",
      "metadata": {
        "delivery_timestamp": "2023-01-01T12:00:05Z"
      }
    }
  ],
  "pagination": {
    "total": 1,
    "limit": 10,
    "offset": 0
  }
}
```

## Response Fields

| Field | Type | Description |
|-------|------|-------------|
| `message_id` | String | Unique identifier for the message |
| `status` | String | Delivery status (e.g., 'delivered', 'failed') |
| `destination_number` | String | The destination phone number |
| `timestamp` | String | When the report was generated |
| `metadata` | Object | Additional delivery information |
| `metadata.delivery_timestamp` | String | When the message was delivered |
| `pagination` | Object | Pagination information |
| `pagination.total` | Integer | Total number of reports available |
| `pagination.limit` | Integer | Number of reports per page |
| `pagination.offset` | Integer | Current offset |

## Error Responses

| Status Code | Description |
|-------------|-------------|
| 400 | Bad Request - Invalid parameters |
| 401 | Unauthorized - Invalid API key or secret |
| 429 | Too Many Requests - Rate limit exceeded |

## Rate Limiting

- **Rate limit**: 100 requests per second per API key
- **Burst limit**: 200 requests per second per API key

## Notes

- Delivery reports are typically available for 7 days after message delivery
- The `status` field indicates the current delivery status of the message
- Use the `metadata` field for additional delivery details specific to the carrier
