# Check Replies

Retrieve replies that have been sent to your account.

<Endpoint
  method={MethodTypes.GET}
  path="/v1/replies"
  auth={false}
  beta={false}
>
  <Method
    type={MethodTypes.GET}
    path="/v1/replies"
    summary="Check replies"
    id="checkReplies"
    auth={false}
  />
</Endpoint>

## Query Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `start_date` | String | No | Filter replies received after this date (ISO 8601 format) |
| `end_date` | String | No | Filter replies received before this date (ISO 8601 format) |
| `limit` | Integer | No | Maximum number of replies to return (default: 100, max: 1000) |
| `offset` | Integer | No | Offset for pagination (default: 0) |
| `destination_number` | String | No | Filter by destination number |
| `source_number` | String | No | Filter by source number |

## Example Request

```bash
curl -X GET \
  'https://api.messagemedia.com/v1/replies?start_date=2023-01-01T00:00:00Z&limit=5' \
  -H 'Authorization: Basic YOUR_API_KEY'
```

## Response Fields

| Field | Type | Description |
|-------|------|-------------|
| `reply_id` | String | Unique identifier for the reply |
| `callback_url` | String | The URL replies are sent to |
| `content` | String | The content of the reply |
| `date_received` | String | When the reply was received (ISO 8601 format) |
| `destination_number` | String | The destination number the reply was sent to |
| `source_number` | String | The source number that sent the reply |
| `metadata` | Object | Additional metadata |
| `metadata.original_message_id` | String | ID of the original message this is a reply to |
| `pagination` | Object | Pagination information |
| `pagination.total` | Integer | Total number of replies available |
| `pagination.limit` | Integer | Number of replies per page |
| `pagination.offset` | Integer | Current offset |

## Error Responses

| Status Code | Description |
|-------------|-------------|
| 400 | Bad Request - Invalid parameters |
| 401 | Unauthorized - Invalid API key or secret |
| 429 | Too Many Requests - Rate limit exceeded |

## Rate Limiting

- **Rate limit**: 100 requests per second per API key
- **Burst limit**: 200 requests per second per API key

## Notes

- Replies are typically available for 7 days after receipt
- To receive replies in real-time, set up a webhook callback URL in your account settings
- The `metadata` field may contain additional information about the reply, such as the original message ID
